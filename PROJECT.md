# 🎓 Student Burnout & Dropout Risk Prediction

End-to-end machine learning portfolio project that explores, models, and
visualizes factors driving student burnout and academic dropout risk.
Built with Python, scikit-learn, XGBoost, SHAP, and Streamlit.

**Dataset**: 800 synthetic student records, 25 features, 2 targets
**License**: CC0 (Public Domain) — free to use and republish

## Quick Start

### Prerequisites

- Python 3.12+
- Windows, macOS, or Linux

### Setup

```bash
cd student-burnout-project

# Create virtual environment
python -m venv .venv

# Activate (Windows)
.venv\Scripts\activate

# Activate (macOS/Linux)
source .venv/bin/activate

# Install dependencies
pip install -r requirements.txt
```

### Run the Notebooks (generates all models, metrics, and plots)

Run in order — `02_Modeling.ipynb` trains 6 models, runs SHAP analysis,
generates plots, and exports trained models to `models/`:

```bash
jupyter notebook notebooks/01_EDA.ipynb       # Exploratory Data Analysis
jupyter notebook notebooks/02_Modeling.ipynb   # Model Training & Evaluation
```

Takes ~2–3 minutes for the modeling notebook. Outputs:
- Classification reports for each model in notebook output
- Confusion matrix heatmaps → `models/*.png`
- ROC curves → `models/roc_curves_binary.png`
- Metrics bar chart → `models/model_comparison_bars.png`
- SHAP beeswarm, dependence, bar plots → `models/shap_*.png`
- Trained models → `models/*.joblib`
- Feature importance rankings → `models/shap_importance.csv`
- Model metrics JSON → `models/model_metrics.json`

### Launch the Interactive Dashboard

```bash
streamlit run app.py
```

Opens in browser at `http://localhost:8501` with 5 pages:

| Page | What It Does |
|------|-------------|
| **Overview** | Dataset stats, key metrics, interactive data table |
| **EDA Explorer** | Select any feature → auto-generated distribution by target |
| **Early Warning System** | Input student attributes → get dropout/burnout predictions |
| **Model Performance** | Side-by-side model comparison, confusion matrices, SHAP |
| **Insights & Recommendations** | Key findings, department breakdowns, interventions |

## Dataset

**Source**: [Kaggle — Harpartap Singh](https://www.kaggle.com/datasets/harpartapsingh13/student-burnout-and-dropout-risk-dataset)

800 student records with 25 columns across 5 categories:

| Category | Columns |
|----------|---------|
| **Demographics** | Age, Gender, Year_of_Study, Department, Residence_Type |
| **Academic** | Attendance_Percent, Study_Hours_Per_Day, Previous_GPA, Backlogs |
| **Lifestyle** | Sleep_Hours, Screen_Time_Hours, Exercise_Freq_Per_Week, Social_Activity_Score, Part_Time_Job |
| **Financial** | Family_Income_Bracket, Financial_Stress_Score, Family_Support_Score |
| **Mental Health** | Stress_Level, Anxiety_Score, Motivation_Score, Peer_Pressure_Score, Counseling_Access |

**Target variables**:
- `Dropout_Risk` — binary (Yes/No) → 437 Yes, 363 No
- `Burnout_Level` — 3-class (Low/Medium/High) → 320 Low, 280 Medium, 200 High

**Missing values**: 612 total across 18 columns — imputed with median (numeric) and mode (categorical).

---

## Modeling Approach

### Preprocessing

1. **Missing values** — median imputation for numeric, mode for categorical
2. **Encoding** — LabelEncoder for all 6 categorical features
3. **Feature engineering** — 3 derived features:
   - `academic_engagement_score` — mean of normalized attendance, study hours, GPA
   - `lifestyle_risk_score` — mean of normalized screen time, inverse sleep, inverse exercise
   - `financial_burden_score` — financial stress normalized, mitigated by family support
4. **Scaling** — StandardScaler applied to all features
5. **Split** — 80/20 train/test, stratified by Dropout_Risk

### Algorithms

| Model | Binary Config | Multiclass Config |
|-------|--------------|-------------------|
| **Logistic Regression** | L2 penalty, max_iter=1000, class_weight=balanced | Same (multinomial) |
| **Random Forest** | GridSearchCV (5-fold): n_estimators∈{100,200,300}, max_depth∈{None,10,20}, min_samples_split∈{2,5}, min_samples_leaf∈{1,2} | Same best params |
| **XGBoost** | GridSearchCV (5-fold): n_estimators∈{100,200,300}, max_depth∈{3,5,7}, learning_rate∈{0.01,0.05,0.1}, subsample∈{0.8,1.0}, colsample_bytree∈{0.8,1.0} | Same best params |

All models use `class_weight="balanced"` or `scale_pos_weight` to handle class imbalance.
GridSearchCV scoring metric: `f1_weighted`.

## Results

### Model Performance

| Model | Target | Accuracy | F1 | Precision | Recall |
|-------|--------|----------|----|-----------|--------|
| Logistic Regression | Dropout Risk | 70.0% | 0.701 | 0.703 | 0.700 |
| **Random Forest** ⭐ | **Dropout Risk** | **71.2%** | **0.713** | **0.713** | **0.712** |
| XGBoost | Dropout Risk | 66.2% | 0.661 | 0.661 | 0.663 |
| **Logistic Regression** ⭐ | **Burnout Level** | **70.0%** | **0.700** | **0.700** | **0.700** |
| Random Forest | Burnout Level | 60.0% | 0.592 | 0.591 | 0.600 |
| XGBoost | Burnout Level | 66.9% | 0.656 | 0.658 | 0.669 |

⭐ = best model per target

> **Note**: All models perform in the 60–71% range, reflecting the inherent
> difficulty of predicting dropout/burnout from this dataset. The strongest
> single feature (`Attendance_Percent`) has only a 0.24 correlation with the
> target, meaning the signal is genuinely weak. The project demonstrates the
> full ML pipeline — all 3 algorithms with hyperparameter tuning — regardless
> of the modest accuracy, which is realistic for this type of problem.

## Technical Notes

- **Pandas 3.x compatibility**: The notebook uses `df[col] = df[col].fillna()`
  instead of `fillna(inplace=True)` due to pandas Copy-on-Write behavior.
- **Data leakage prevention**: `Dropout_Risk_enc` and `Burnout_Level_enc` are
  excluded from the feature matrix to avoid target leakage.
- **Small test set**: With 160 test samples, accuracy values are multiples of
  1/160 (0.625%). Round-looking numbers like 0.7000 = 112/160 are expected.
